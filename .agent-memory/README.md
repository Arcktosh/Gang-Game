# Agent Memory

> Generated by `pnpm agent:memory`. Do not edit manually. Source fingerprint: `e984395c02aabf70`.

This directory is a compact, deterministic retrieval layer for coding agents. Start with the smallest relevant index instead of scanning the entire repository.

## Retrieval order

1. Read `../AGENTS.md` for operating rules and repository boundaries.
2. Read `tasks.json` and select one task whose dependencies are satisfied.
3. Use `manifest.md` to identify the owning workspace and commands.
4. Use `public-api.md` and `symbols.md` to find stable entrypoints.
5. Use `imports.md` before changing shared modules or dependency direction.
6. Use `files.md` only when broader discovery is required.

## Files

- `files.md` — path, role, size, scope, and export counts.
- `imports.md` — workspace dependencies, fan-in/fan-out, and module-level imports.
- `manifest.md` — package manifests, scripts, aliases, and environment key names.
- `public-api.md` — workspace exports, Next.js API routes, and pages.
- `symbols.md` — top-level declarations and export locations.
- `tasks.json` — manually curated, machine-readable work queue.
- `tasks.schema.json` — JSON Schema for the work queue.

## Maintenance

Run `pnpm agent:memory` after changing source structure, imports, exports, manifests, routes, or symbols. Run `pnpm agent:memory:check` to verify that generated memory and the task queue are valid.
